# OniT — полный план запуска

От фотографий до Stripe и Telegram-бота. Всё по порядку.

**Легенда:**
- 🧍 **[Ты]** — делаешь сам (аккаунты, сервер, секреты, фото).
- 🤖 **[Я могу]** — могу сделать в коде, когда скажешь.
- ⚠️ **[Проверить]** — не предполагаем, а проверяем (юридика/возраст).

> **Архитектура простыми словами:**
> 1. Сайт (Astro) — статика, лежит на сервере Oracle, отдаётся через Caddy с HTTPS.
> 2. Маленький Node-сервис («бот-сервер») на том же сервере — принимает webhook от Stripe и пишет тебе в Telegram.
> 3. Stripe шлёт webhook на `https://sudonit.com/stripe/webhook` при каждой оплате.
> 4. Telegram-бот получает сообщение от Node-сервиса и пишет тебе: «куплено X на Y€».
>
> Деньги, карты, 3D-Secure — всё держит Stripe. Мы храним 0 платёжных данных.

  - Ремонт — сейчас по модели «оплата после работы, лично». Через Stripe это скорее **счёт/ссылка на оплату**, которую ты шлёшь вручную после починки, а не кнопка «купить» на сайте.
  - Очки / English — пока не продаются.
  - **Вывод:** реши, где на сайте появляются кнопки оплаты, а где остаётся «напишите в WhatsApp».
- [ ] 🧍 **Способ интеграции Stripe** (см. Фазу 3):
  - **Вариант A (рекомендую для старта):** Stripe **Payment Links** — просто ссылки, ноль кода на сайте, webhook → Telegram. Запускается за вечер.
  - **Вариант B:** кнопки Checkout прямо на сайте (нужна небольшая доработка кода — 🤖 это делаю я).
- [x] ✅ **Возраст и Stripe — решено.** Юридически и финансово всё оформляется на **взрослого** (он — владелец Stripe-аккаунта и autónomo), Александр выполняет саму работу. Stripe требует 18+ — этим закрыто. Чтобы прошло верификацию без сюрпризов, данные должны сходиться в одну цепочку:
  - Stripe проверяет **личность владельца** (DNI/паспорт) — аккаунт оформлен на этого взрослого по-настоящему.
  - **IBAN для выплат** — его; autónomo — в идеале на нём же (всё в одном лице).
  - Счета/чеки клиентам — от его имени; в Aviso legal он указан как ответственный.
  - Для клиента и закона «продавец» = этот взрослый, Александр = исполнитель.
- [ ] 🧍 **Валюта/регион:** EUR, Испания. Подключить **Bizum** в Stripe — местные клиенты любят его больше карт.

## Фаза 2 — Сервер Oracle (бесплатный) + домен + HTTPS

Цель: сайт открывается по `https://sudonit.com`.

- [ ] 🧍 **Создать инстанс.** Oracle Cloud → Always Free → **Ampere A1 (ARM)**, Ubuntu 24.04. Бери, например, 1–2 OCPU / 6–12 GB RAM — это в пределах бесплатного. (ARM = arm64, Node там работает нормально.)
- [ ] 🧍 **Зарезервировать публичный IP** (Reserved, не Ephemeral) — чтобы не менялся.
- [ ] 🧍 **Открыть порты.** Это главная ловушка Oracle — порты надо открыть в ДВУХ местах:
  1. **Security List / NSG** в облаке: ingress TCP **80**, **443** (22 уже открыт).
  2. **Внутри ВМ** (Ubuntu от Oracle имеет свои iptables-правила):
     ```bash
     sudo iptables -I INPUT 6 -m state --state NEW -p tcp --dport 80 -j ACCEPT
     sudo iptables -I INPUT 6 -m state --state NEW -p tcp --dport 443 -j ACCEPT
     sudo netfilter-persistent save
     ```
- [ ] 🧍 **DNS.** У регистратора домена `sudonit.com`:
  - `A`-запись `@` → IP сервера.
  - `A`-запись `www` → тот же IP (или CNAME на корень).
- [ ] 🧍 **Базовая защита сервера:**
  ```bash
  sudo apt update && sudo apt upgrade -y
  sudo apt install -y fail2ban
  # вход по SSH-ключу, пароль-логин отключить (PasswordAuthentication no в sshd_config)
  ```
- [ ] 🧍 **Caddy** (веб-сервер + автоматический HTTPS Let's Encrypt):
  ```bash
  sudo apt install -y debian-keyring debian-archive-keyring apt-transport-https curl
  curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/gpg.key' | sudo gpg --dearmor -o /usr/share/keyrings/caddy-stable-archive-keyring.gpg
  curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/debian.deb.txt' | sudo tee /etc/apt/sources.list.d/caddy-stable.list
  sudo apt update && sudo apt install -y caddy
  ```
- [ ] 🧍 **Каталог сайта:** `sudo mkdir -p /var/www/onit && sudo chown -R $USER:$USER /var/www/onit`
- [ ] 🧍 **`/etc/caddy/Caddyfile`** (статика + проксирование webhook на Node-сервис из Фазы 4):
  ```caddy
  sudonit.com, www.sudonit.com {
      encode zstd gzip
      root * /var/www/onit
      file_server

      # Stripe webhook → локальный Node-сервис (Фаза 4)
      handle /stripe/webhook {
          reverse_proxy 127.0.0.1:8787
      }
  }
  ```
  Затем: `sudo systemctl reload caddy`
- [ ] 🤖🧍 **Деплой статики.** Локально собираешь и заливаешь готовую папку:
  ```bash
  npm run build
  rsync -avz --delete dist/ ubuntu@<IP>:/var/www/onit/
  ```
  🤖 Могу положить в репозиторий готовый `deploy.sh`, чтобы это была одна команда.
- [ ] ✅ **Проверка:** `https://sudonit.com` открывается, замок (HTTPS) есть, языки/страницы работают.

## Фаза 4 — Node-сервис (webhook → Telegram)

Маленький сервис на сервере: принимает событие Stripe, проверяет подпись, шлёт тебе в Telegram. 🤖 **Это я напишу целиком, когда попросишь** — здесь только чек-лист и важные нюансы.

- [ ] 🧍 Установить Node на сервере (LTS, arm64): через `nvm` или `apt`.
- [ ] 🤖 Сервис (Express/Fastify) с одним маршрутом `POST /stripe/webhook`:
  - ⚠️ **Критично:** для проверки подписи нужен **сырой (raw) body**, а не распарсенный JSON (`express.raw({ type: 'application/json' })`). Иначе подпись не сойдётся.
  - Проверка: `stripe.webhooks.constructEvent(rawBody, sig, STRIPE_WEBHOOK_SECRET)`.
  - На `checkout.session.completed` — собрать сумму/товар/email и отправить в Telegram.
  - Отвечать Stripe `200` быстро (тяжёлое — в фон).
- [ ] 🧍 **Секреты** в `/etc/onit-bot.env` (права `chmod 600`):
  ```
  STRIPE_SECRET_KEY=sk_test_...
  STRIPE_WEBHOOK_SECRET=whsec_...
  TELEGRAM_BOT_TOKEN=...
  TELEGRAM_CHAT_ID=...
  PORT=8787
  ```
- [ ] 🤖🧍 **systemd-юнит** `/etc/systemd/system/onit-bot.service` (`Restart=always`, `EnvironmentFile=/etc/onit-bot.env`), затем:
  ```bash
  sudo systemctl enable --now onit-bot
  ```
