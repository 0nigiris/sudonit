* Product storytelling

* Huge typography
* Cinematic presentation

* User almost never wonders where to click

# CURRENT LESSONS LEARNED

The Orbit version taught us what does NOT work.

## We do NOT want

* Orbit as primary navigation
* Ctrl+K command palette
* Hidden interactions
* Operating-system simulations
* Dashboard metaphors
* Interface-first design
* Panels replacing pages
* User learning a custom navigation system

## Why

Visitors should focus on Alexander.

Not on understanding the website.

# VISUAL PRINCIPLES

1. Person first.
2. Story second.
3. Projects third.
4. Interface last.

The website should explain Alexander.

The website should not explain itself.

# WOW MOMENTS

Only one major wow moment is needed.

Current idea:

A project showcase section where project cards rotate around a circular path.

Important:

* Viewed from an angled front perspective
* Not top-down
* Cards have depth
* Front cards appear larger
* Rear cards appear smaller
* Scroll controls rotation
* After one rotation, normal scrolling resumes

This section exists to showcase projects.

Not to navigate the entire website.
