# CLAUDE.md

## Project

Personal website for Alexander (OniT).

This is NOT a developer portfolio.

This is NOT a dashboard.

This is NOT an operating-system simulation.

This is NOT an experimental UI showcase.

The website exists to introduce Alexander, his work, his projects, and his services.

---

## Core Rule

Alexander first.

Projects second.

Interface third.

If a design decision makes the interface more important than Alexander, remove it.

---

## User Experience

The primary interaction is scrolling.

The visitor should never need to learn the interface.

The visitor should never wonder:

"What am I supposed to click?"

The experience should feel natural and obvious.

---

## Navigation Philosophy

Homepage:

* Introduction
* About
* Story
* Projects
* Contact

Project pages:

* Repair
* Tutoring
* English Project
* Smart Glasses Project

Each project has its own page.

No floating panels.

No hidden navigation.

No command palettes.

No keyboard shortcuts.

No Ctrl+K.

---

## Design Philosophy

Inspired by:

* Apple
* Nothing
* Modern product storytelling
* High-end editorial websites

Not inspired by:

* Terminal UIs
* Dashboard UIs
* Admin panels
* Futuristic operating systems

---

## Animation Philosophy

Animations should support content.

Animations should never become the content.

Good animation:

* fade up
* reveal on scroll
* scale transitions
* sticky sections
* smooth transforms

Bad animation:

* constant movement
* distracting motion
* UI gimmicks

If animations are removed, the website should feel worse.

If animations become the main thing people notice, they are too strong.

---

## Development Workflow

1. Research
2. Wireframes
3. User flow
4. Visual concepts
5. Design review
6. Implementation plan
7. Code

Never skip directly to coding.

Always validate the concept first.

---

## Important

When unsure:

Choose clarity over cleverness.

Choose storytelling over interface.

Choose simplicity over novelty.
